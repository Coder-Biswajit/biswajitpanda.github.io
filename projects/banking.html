Overview

The Banking Management System is a backend-focused application simulating core banking operations — account creation, deposits and withdrawals, fund transfers, and loan management. Designed for both customers and admins with role-based access control and data validation at every layer.

The project prioritizes data integrity and security: every transaction is validated, authenticated, and logged. Analytics reports were generated using SQL and exported via Pandas, then visualized in Power BI dashboards for executive-level summaries.

Exported data via Pandas into Power BI — visual summaries of branch performance, customer segments, and loan status.
Technical Approach

The application is built entirely in Python, with a CLI interface connecting to a MySQL relational database via the mysql-connector-python library. Each entity — customers, accounts, loans, branches — is modeled as a normalized table with proper primary and foreign key constraints.

Transaction processing uses parameterized SQL queries to prevent injection attacks. Analytics queries use window functions, GROUP BY, and HAVING clauses to compute branch-wise summaries and loan performance metrics. Output data is exported through Pandas DataFrames for further Power BI visualization.

Learnings

This project gave me deep hands-on experience with relational database design principles — normalization, referential integrity, and query optimization. Writing complex multi-table queries to surface analytics forced me to think about data structure before writing a single line of Python.

Exporting to Power BI showed me how the same underlying data can tell very different stories depending on how it's presented — and why the last mile of communication matters as much as the analysis itself.
