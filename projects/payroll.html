Overview

The Employee Payroll System is a data-first HR solution that handles the full payroll lifecycle — from employee records and attendance to salary computation with overtime, bonuses, and deductions. The focus was on building a robust, normalized database that could support complex analytical queries without redundancy.

Beyond operations, the system surfaces department-wise payroll insights and cost distribution analytics, helping HR teams understand where salary budget is allocated and identify anomalies in compensation data.

Key Features
🗄️
Normalized Schema
Fully normalized MySQL database with primary keys, foreign keys, and constraints across employees, departments, payroll, and deductions tables.
🧮
Salary Computation
Automated calculation of base salary, overtime pay, performance bonuses, tax deductions, and net take-home with validated data handling.
🏢
Department Analytics
Complex SQL with JOIN, GROUP BY, and HAVING to compute department-wise payroll totals, averages, and outlier detection.
📊
Power BI Cost Dashboard
Visual cost distribution breakdown by department, role tier, and pay component — enabling executive-level HR decisions.
Technical Approach

The database was designed following third normal form (3NF) — eliminating transitive dependencies to ensure that payroll data remains consistent even as employee records change. Salary component logic (overtime multipliers, deduction brackets) is implemented as Python-computed values written back to the database, keeping business logic cleanly separated from storage.

Analytics queries use aggregate functions with conditional logic — CASE statements inside SUM to compute department cost by salary tier. Power BI connects directly to the MySQL database and refreshes dashboards from live data, providing a real-time view of the organization's payroll posture.

Learnings

Building this system taught me that data modeling is the most critical phase of any database project — poor schema design creates analytical debt that compounds over time. Spending extra time on normalization upfront made every subsequent query simpler and more reliable.

Working with payroll data also introduced me to the complexity of financial calculations in software — edge cases like partial months, retroactive adjustments, and rounding rules that seem simple on paper but require careful implementation in code.
